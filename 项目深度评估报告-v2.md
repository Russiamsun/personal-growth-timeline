## 三、仍需改进的问题

### 🔴 高优先级问题

#### 3.1 缺少用户认证机制

**问题描述**:
- 当前Supabase使用anon key + 公开RLS策略
- 任何人都可以读写数据，无用户隔离
- 不适合多用户场景（如其他孩子使用）

**影响**: 安全风险，数据可被他人修改或删除

**建议方案**:
```typescript
// 1. 启用Supabase Auth
import { supabase } from '@/lib/supabase';

// 登录
const signIn = async () => {
  const { data, error } = await supabase.auth.signInWithPassword({
    email: 'emma@example.com',
    password: 'password',
  });
};

// 2. 修改RLS策略
CREATE POLICY "Users can only access their own data"
ON activities FOR ALL
USING (auth.uid() = user_id);

// 3. 数据表添加user_id字段
ALTER TABLE activities ADD COLUMN user_id UUID REFERENCES auth.users();
```

#### 3.2 缺少错误监控和日志系统

**问题描述**:
- 仅有console.log/error，无结构化日志
- 生产环境无法追踪错误
- 用户操作无审计日志

**建议方案**:
```typescript
// 1. 集成Sentry错误监控
import * as Sentry from "@sentry/react";

Sentry.init({
  dsn: "your-dsn",
  integrations: [new Sentry.BrowserTracing()],
});

// 2. 结构化日志
const logger = {
  info: (message: string, context?: object) => {
    console.log(`[INFO] ${message}`, context);
    // 可选：发送到日志服务
  },
  error: (message: string, error: Error) => {
    console.error(`[ERROR] ${message}`, error);
    Sentry.captureException(error);
  }
};
```

#### 3.3 表单页面仍有代码重复

**问题描述**:
- CreateActivityPage/EditActivityPage等6个表单页面存在相似逻辑
- 双语言输入处理重复
- 表单验证逻辑分散

**建议方案**:
```typescript
// 1. 抽取通用表单Hook
const useActivityForm = (initialData?: Activity) => {
  const [formData, setFormData] = useState(initialData || defaultActivity);
  const [errors, setErrors] = useState<Record<string, string>>({});
  
  const validate = () => { /* 通用验证 */ };
  const handleSubmit = () => { /* 通用提交 */ };
  
  return { formData, setFormData, errors, validate, handleSubmit };
};

// 2. 表单字段组件化
<FormField
  name="titleZh"
  label="中文标题"
  required
  value={formData.titleZh}
  onChange={(v) => setFormData({ ...formData, titleZh: v })}
  error={errors.titleZh}
/>
```
