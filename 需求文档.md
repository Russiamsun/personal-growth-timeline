# 个人成长时间轴网站需求文档

## 一、项目概述

**项目名称**: 个人成长时间轴网站
**项目定位**: 个人成长时光轴记录平台,按时间线整理人生经历
**核心价值**: 记录人生每一段经历,保存珍贵回忆,见证成长轨迹
**目标用户**: 希望系统化记录个人成长经历、留存人生回忆的用户

---

## 二、目标用户分析

| 用户类型 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 学生群体 | 18-25岁,追求学业进步 | 记录学习成长、校园生活 |
| 职场新人 | 25-30岁,职业发展初期 | 记录职场经历、技能提升 |
| 自我提升者 | 30岁以上,追求全面发展 | 记录人生感悟、家庭时光 |
| 创业者/自由职业者 | 需要自我驱动 | 记录创业历程、重要节点 |

---

## 三、核心功能模块

| 优先级 | 功能模块 | 功能描述 | 用户价值 |
|--------|---------|---------|---------|
| **P0** | 时间流水记录 | 创建记录、添加文字+照片+心情 | 核心记录功能 |
| **P0** | 年份时间轴 | 按年份展示所有经历 | 时间线浏览 |
| **P0** | 图片管理 | 上传多张照片、预览、排序 | 丰富内容呈现 |
| **P1** | 心情标记 | 选择心情图标/标签 | 情绪记录 |
| **P1** | 搜索筛选 | 按时间、关键词、心情筛选 | 快速查找 |
| **P2** | 导出分享 | 导出PDF/生成分享链接 | 留存纪念 |

## 四、时间流水记录详细设计

### 4.1 单条记录字段设计

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 标题 | 文本 | 是 | 这段经历的简短标题 |
| 发生时间 | 日期 | 是 | YYYY-MM-DD(可精确到时分) |
| 内容描述 | 富文本 | 是 | 详细记录这段经历 |
| 照片 | 图片数组 | 否 | 支持1-9张照片 |
| 心情 | 枚举 | 否 | 开心/平静/感动/兴奋/难过/焦虑等 |
| 标签 | 文本数组 | 否 | 自定义标签(如#旅行 #工作 #学习) |
| 地点 | 文本 | 否 | 发生地点 |
| 创建时间 | 时间戳 | 自动 | 记录创建时间 |
| 更新时间 | 时间戳 | 自动 | 最后修改时间 |

### 4.2 时间轴展示方式

| 视图类型 | 说明 |
|---------|------|
| **年份视图** | 按年份分组,显示年度概览卡片 |
| **月份视图** | 点击某年展开月份列表 |
| **列表视图** | 按时间倒序展示所有记录 |
| **卡片视图** | 图文并茂的瀑布流展示 |

### 4.3 心情系统设计

| 心情类别 | 图标建议 | 使用场景 |
|---------|---------|---------|
| 😊 开心 | 笑脸 | 快乐、喜悦的时刻 |
| 😌 平静 | 平静脸 | 日常、平淡的经历 |
| 🥰 感动 | 爱心眼 | 温暖、感动的瞬间 |
| 🎉 兴奋 | 庆祝 | 重大成就、好消息 |
| 😢 难过 | 哭脸 | 失落、悲伤的时刻 |
| 😰 焦虑 | 焦虑脸 | 压力、担忧的时刻 |
| 🤔 思考 | 思考脸 | 反思、成长的时刻 |
| 💪 努力 | 握拳 | 奋斗、拼搏的时光 |

## 五、功能页面设计

### 5.1 主要页面结构

| 页面名称 | 功能说明 |
|---------|---------|
| **时间轴首页** | 年份列表 + 每年精选记录预览 |
| **年份详情页** | 该年份所有记录的时间线展示 |
| **记录详情页** | 单条记录的完整展示(文字+照片+心情) |
| **创建/编辑记录** | 表单页面,支持上传照片、选择心情 |
| **搜索页面** | 全局搜索、筛选功能 |

### 5.2 年度概览卡片设计

每个年份卡片展示以下元素:

| 元素 | 内容 |
|------|------|
| 年份标题 | 如"2024年" |
| 记录数量 | "共记录 XX 段经历" |
| 心情统计 | 该年度心情分布(如:开心60%、平静20%...) |
| 精选照片 | 展示3-4张代表性照片(缩略图) |
| 关键词标签 | 该年度高频标签云 |
| 进入按钮 | "查看2024年全部记录 →" |

### 5.3 用户交互流程

#### 创建记录流程

1. 点击"新建记录"按钮
2. 填写基本信息
   - 输入标题
   - 选择发生时间(日期选择器)
   - 选择心情(心情图标)
   - (可选)输入地点
   - (可选)添加标签
3. 编写内容
   - 富文本编辑器
   - 支持加粗、斜体、列表等格式
4. 上传照片
   - 拖拽或点击上传
   - 支持批量上传
   - 可调整顺序、删除
   - 预览缩略图
5. 保存发布
   - 保存到数据库
   - 自动跳转到详情页

#### 浏览时间轴流程

1. 进入首页,看到年份列表
2. 点击某个年份,进入该年详情
3. 时间线展示该年所有记录
   - 按时间倒序排列
   - 左侧时间轴,右侧卡片
4. 点击某条记录,查看详情
   - 大图展示照片
   - 完整文字内容
   - 心情、标签等元信息
5. (可选)编辑或删除该记录

## 六、数据存储设计

### 6.1 数据库表结构

**记录表(records)**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 主键 | 记录唯一标识 |
| title | 文本 | 标题 |
| content | 富文本 | 内容描述 |
| record_date | 日期 | 发生日期 |
| mood | 枚举 | 心情 |
| location | 文本 | 地点 |
| tags | 数组 | 标签 |
| photos | 数组 | 照片URL |
| year | 整数 | 年份(冗余字段便于查询) |
| created_at | 时间戳 | 创建时间 |
| updated_at | 时间戳 | 更新时间 |

**照片表(photos)**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 主键 | 照片唯一标识 |
| record_id | 外键 | 关联记录ID |
| url | 文本 | 照片URL |
| order | 整数 | 排序序号 |
| uploaded_at | 时间戳 | 上传时间 |

### 6.2 文件存储方案

| 方案 | 适用场景 | 说明 |
|------|---------|------|
| 本地存储 | 开发/测试 | 存储在服务器本地磁盘 |
| 云存储(推荐) | 生产环境 | 阿里云OSS/腾讯云COS/七牛云 |
| 第三方图床 | 低成本方案 | SM.MS、ImgBB等免费图床 |
