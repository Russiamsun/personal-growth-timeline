## 三、待完善的问题

### 🔴 高优先级问题

#### 1. 数据存储方案存在隐患

**问题描述**：
- 当前使用localStorage存储所有数据（包括照片的Base64）
- localStorage有5-10MB限制，照片数据会快速耗尽空间
- 已有QuotaExceededError处理，但用户体验不佳

**影响**：用户上传多张照片后可能无法保存数据

**建议**：
- 集成Supabase（项目已安装依赖但未使用）进行云端存储
- 照片使用云存储（如Supabase Storage、Cloudinary）
- 实现离线优先策略 + 后台同步

```typescript
// 建议的数据存储策略
// 1. 小型数据 → localStorage（快速访问）
// 2. 大型数据 → IndexedDB（更大容量）
// 3. 云端同步 → Supabase（持久化）
```

#### 2. 富文本编辑功能缺失

**问题描述**：
- 数据模型中`content`字段定义为HTML富文本
- 但表单使用普通`<textarea>`，无法输入格式化内容
- 详情页使用`dangerouslySetInnerHTML`渲染，存在XSS风险

**影响**：用户无法创建格式丰富的内容，安全风险

**建议**：
- 集成富文本编辑器（推荐 TipTap 或 React-Quill）
- 添加XSS过滤（使用 `DOMPurify` 库）

#### 3. 缺少数据备份与恢复功能

**问题描述**：
- 数据仅存储在浏览器本地
- 用户清除浏览器数据后会丢失所有记录
- 无法跨设备访问

**影响**：数据安全性和可迁移性差

**建议**：
- 实现数据导出功能（JSON/PDF格式）
- 添加数据导入功能
- 实现云账户系统（可选）

### 🟠 中优先级问题

#### 4. 表单组件存在大量重复代码

**问题描述**：
- `CreateActivityPage`、`EditActivityPage`、`CreateQuestionPage`、`EditQuestionPage`、`CreateReflectionPage`、`EditReflectionPage` 六个页面代码高度相似
- 双语输入模式切换逻辑在每个页面重复实现

**影响**：维护成本高，修改需要同步多处

**建议**：
```typescript
// 抽取通用表单组件
// 1. BilingualInputField - 双语输入字段组件
// 2. InputModeSelector - 输入模式选择器
// 3. FormField - 通用表单字段组件（带验证）
// 4. BaseFormPage - 基础表单页面HOC
```

#### 5. 时间轴页面性能优化

**问题描述**：
- 使用`setInterval`逐个显示活动（150ms间隔）
- 活动数量多时会导致长时间动画
- 每次渲染都重新计算分组和排序

**建议**：
```typescript
// 使用 useMemo 缓存计算结果
const sortedActivities = useMemo(() => 
  [...activities].sort((a, b) => 
    new Date(b.date).getTime() - new Date(a.date).getTime()
  ), [activities]);

const groupedActivities = useMemo(() => 
  // 分组逻辑
, [sortedActivities]);
```

#### 6. 图片加载错误处理不完善

**问题描述**：
- PhotoUploader组件中的错误处理使用`alert()`
- 图片加载失败时显示SVG占位图，但用户体验不佳

**建议**：
- 使用Toast通知替代alert
- 添加图片上传进度显示
- 实现图片上传重试机制

#### 7. 缺少表单自动保存功能

**问题描述**：
- 用户填写长表单时，意外刷新页面会丢失所有输入
- 没有草稿保存机制

**建议**：
- 实现表单数据的本地自动保存
- 添加"保存草稿"功能

  useEffect(() => {
    const timer = setInterval(() => {

    }, interval);
    return () => clearInterval(timer);
  }, [data, key, interval]);
};
